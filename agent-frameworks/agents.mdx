Anchor Browser includes three pre-configured AI agents for browser automation. Choose the one that best fits your workflow:

## Available Agents

### Browser-Use (Default)

<img src="/images/agent-browser-use-icon-light.svg" alt="Browser-Use" width="80" noZoom className="float-right ml-3 block dark:hidden" />
<img src="/images/agent-browser-use-icon-dark.svg" alt="Browser-Use" width="80" noZoom className="float-right ml-3 hidden dark:block" />
Browser-Use is a versatile fast, DOM based, autonomous browser agent with pause/resume, human-in-the-loop controls, and structured outputs. It supports multiple providers including OpenAI (and Azure OpenAI), Gemini, and Groq, making it ideal for complex, multi-step workflows.
It is a fork of the [browser-use open-source repository](https://github.com/browser-use/browser-use) 

### OpenAI CUA 
<img src="/images/agent-openai-light.svg" alt="OpenAI CUA" width="80" noZoom className="float-right ml-3 block dark:hidden" />
<img src="/images/agent-openai-dark.svg" alt="OpenAI CUA" width="80" noZoom className="float-right ml-3 hidden dark:block" />
OpenAI’s Computer Use Agent offers high-accuracy, screenshot-driven interactions for direct, task-focused automation. Provider: OpenAI only.

### Gemini Computer Use
<img src="/images/agent-gemini-color.png" alt="Gemini Computer Use" width="80" noZoom className="float-right ml-3" />
Google’s Gemini Computer Use agent enables multi-turn interactions and structured outputs tightly integrated with the Gemini platform. Provider: Google Gemini only.

### Other Playwright-Based Agent
Anchor Browser supports any Playwright-based agent, so if you prefer a different agent than the three above, you can integrate your own custom agent.
## Quick Start

<Accordion title="Code example - Using Agents" defaultOpen>
<CodeGroup>
```python python
import os
from anchorbrowser import Anchorbrowser

anchor_client = Anchorbrowser(api_key=os.getenv("ANCHOR_API_KEY"))

# Use different agents with task_options
result = anchor_client.agent.task(
    "Navigate to example.com and extract the page title",
    task_options={
        "url": "https://example.com",
        "output_schema": {"type": "object", "properties": {"title": {"type": "string"}}}
    }
)

# For specific agents, use tools/perform_web_task
result = anchor_client.tools.perform_web_task(
    session_id="your-session-id",
    prompt="Fill out the contact form",
    agent="openai-cua"
)
```

```typescript typescript
import { Anchorbrowser } from 'anchorbrowser';

const anchorClient = new Anchorbrowser({ 
  apiKey: process.env.ANCHOR_API_KEY 
});

(async () => {
  // Use different agents with taskOptions
  const result = await anchorClient.agent.task(
    "Navigate to example.com and extract the page title",
    {
      url: "https://example.com",
      outputSchema: { type: "object", properties: { title: { type: "string" } } }
    }
  );

  // For specific agents, use tools/perform_web_task
  const result2 = await anchorClient.tools.performWebTask({
    sessionId: "your-session-id",
    prompt: "Fill out the contact form",
    agent: "gemini-computer-use"
  });
})();
```

## Agent Configuration

### agent.task() Parameters
| Parameter | Type | Description |
|-----------|------|-------------|
| `url` | string | Target URL to navigate to |
| `output_schema` / `outputSchema` | object | JSON schema for structured output |
| `on_agent_step` / `onAgentStep` | function | Callback for step-by-step execution logs |

### tools.perform_web_task() Parameters
| Agent | Required | Optional |
|-------|----------|----------|
| **browser-use** | `prompt` | `agent`, `url`, `provider`, `model`, `max_steps`, `highlight_elements`, `detect_elements`, `human_intervention` |
| **openai-cua** | `prompt` | `agent`, `url` |
| **gemini-computer-use** | `prompt` | `agent`, `url` |
