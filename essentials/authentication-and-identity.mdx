Anchor allows you to save an existing browser state as "profiles" for use in future browser sessions. This feature enables users to:

- Store authenticated sessions and identities, allowing to stay logged in to websites
- Improve overall speed and performance

The following guide explains how to create and use Identity Profiles in Anchor Browser.

## Quick start - Create and use a profile




<Steps>
<Step title="Start a playground session with a proflie">
Through the Anchor playground, create a profile on the configuration area. Then, click to start a session.
##### **OR**
[create-session via SDK](/sdk-reference/browser-sessions/start-browser-session) (or [API](/api-reference/browser-sessions/start-browser-session))
while configuring the new profile to persist

<CodeGroup>
```JavaScript node.js
import { Anchorbrowser } from "anchorbrowser";

(async () => {
 const anchor_client = new Anchorbrowser()
 const session = await anchor_client.sessions.create({
    browser: {
        profile: {
            name: 'new-profile',
            persist: true
            }
        }
    })
 console.log(session)
})();
```
```python python
from anchorbrowser import Anchorbrowser
import os

anchor_client = Anchorbrowser(api_key=os.getenv("ANCHOR_API_KEY"))
session = anchor_client.sessions.create(browser={
    "profile": {
        "name": "new-profile",
        "persist": True
    }
})
print(session)
```
</CodeGroup>

</Step>
<Step title="Authenticate Once">
Authenticate to the target service using the playground or the Live URL of the session to create a browser context with the required cookies and session data.
</Step>
<Step title="Save Profile">
#### Playground
Save the profile using the 'Save Profile' button in the Anchor Browser Playground.
#### Live URL
End the session using the [Session History page](https://app.anchorbrowser.io/sessions).
<Note>This operation will end the current playground browser session</Note>
</Step>
<Step title="Use the profile in other sessions">
Now, when using the [create session](/sdk-reference/browser-sessions/start-browser-session) API, pass the `profile` parameter with the name of the profile you created to load the saved browser context.

<CodeGroup>
```JavaScript node.js
import { Anchorbrowser } from "anchorbrowser";

(async () => {
 const anchor_client = new Anchorbrowser()
 const session = await anchor_client.sessions.create({
    browser: {
        profile: {
            name: 'new-profile'
        }
    }
    })
 console.log(session)
})();
```
```python python
from anchorbrowser import Anchorbrowser
import os

anchor_client = Anchorbrowser(api_key=os.getenv("ANCHOR_API_KEY"))
session = anchor_client.sessions.create(browser={
    "profile": {
        "name": "new-profile"
    }
})
print(session)
```
</CodeGroup>
</Step>
</Steps>

For additional capabilities and advanced use cases, refer to the API documentation for [profiles](/api-reference/profiles).
