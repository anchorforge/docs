## Authenticated Task

For tasks that require login to an application. Make sure you have set up the required [authentication and identity](/essentials/authentication-and-identity).

### Create a Task

```bash cURL
curl -X POST "https://api.anchorbrowser.io/v1/tools" \
  -H "anchor-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Send LinkedIn Message",
    "description": "Send a message to a colleague on LinkedIn",
    "application_id": "your-application-uuid",
    "input_schema": [
      {
        "display_name": "Recipient",
        "type": "string",
        "required": true,
        "description": "Name or profile URL of the recipient"
      },
      {
        "display_name": "Message",
        "type": "string",
        "required": true,
        "description": "The message content to send"
      }
    ]
  }'
```

### Run a Task

```bash cURL
curl -X POST "https://api.anchorbrowser.io/v2/tasks/{taskId}/run" \
  -H "anchor-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "input_params": {
      "Recipient": "John Doe",
      "Message": "Hello, how are you?"
    },
    "identity_id": "your-identity-uuid"
  }'
```

<Warning>
The browser session is not terminated by default when a task ends. To automatically close the session, set `"cleanup_sessions": true` in the request body.
</Warning>

<br />
See [Getting Run Results](#getting-run-results) to poll for the task output.
<br />
<br />

## Getting Run Results

Whether running **authenticated** or **unauthenticated** tasks, the run endpoint returns a `run_id`. Use it to poll for the result:

```bash cURL
curl "https://api.anchorbrowser.io/v2/tasks/runs/{runId}/status" \
  -H "anchor-api-key: YOUR_API_KEY"
```

The response includes:
- `status`: `queued`, `running`, `success`, `failure`, `timeout`, or `cancelled`
- `result`: the task output (when status is `success`)
- `error`: error message (when status is `failure`)

# Legacy Tasks API

The Legacy Tasks API enables you to **create, version, and execute reusable browser automation code** programmatically in your Anchor Browser sessions.

<img
  className="mx-auto"
  src="/images/tasks-legacy-dashboard.webp"
  alt="Legacy tasks dashboard"
/>

<Note>
Legacy tasks are executed in a secure sandbox environment and can access the full Anchor Browser API for automation capabilities.
</Note>

## Writing Legacy Task Code

For reliable execution, **follow these guidelines:**
 - Write your **code in TypeScript.**
 - **Export** a single **default async function.**
 - In that function, **return** whatever your workflow requires as **output** (e.g., status, messages, domain data).

<Note>
Legacy tasks can receive values as inputs. **All input names must be prefixed with `ANCHOR_`**
</Note>

### Basic Legacy Task Example

<CodeGroup>
```typescript typescript
import AnchorClient from 'anchorbrowser';

// Initialize the Anchor client with your API key
const anchorClient = new AnchorClient({
    apiKey: process.env.ANCHOR_API_KEY,
});

// Export the main function as the default export
export default async function run() {

    // Create a new browser instance
    const browser = await anchorClient.browser.create();
    const page = browser.contexts()[0].pages()[0];

    // Access input values
    const targetUrl = process.env.ANCHOR_TARGET_URL;
    const maxPages = parseInt(process.env.ANCHOR_MAX_PAGES || '10');

    // Implement your automation logic
    await page.goto(targetUrl);
    console.log(`Scraping up to ${maxPages} pages from ${targetUrl}`);

    // Always close the browser when done
    await browser.close();

    // Return a result object with success status and message
    return {
        success: true,
        message: 'Task completed successfully'
    };
}
```
</CodeGroup>

## Async Legacy Task

By default, legacy task execution is **synchronous** - the API call waits for the task to complete before returning results. For long-running tasks, you can use **asynchronous execution** to start the task and check results later.

<img
  className="mx-auto"
  src="/images/tasks-legacy-run.webp"
  alt="Legacy task run interface"
  width="600"
/>

### Running Legacy Tasks Asynchronously

When you set `async: true`, the API returns immediately with a confirmation that the legacy task has started. You can then poll for execution results using the task execution history endpoint.

<CodeGroup>
```typescript node.js
// Run the task asynchronously
const execution = await client.task.run({
  taskId: taskId,
  version: '1',
  async: true, // Enable async execution
  inputs: {
    ANCHOR_TARGET_URL: 'https://example.com',
    ANCHOR_MAX_PAGES: '10'
  }
});

console.log('Task execution started:', execution.data);
// Response: { async: true, success: true, message: 'Task execution started', taskId: '...' }
```

```python python
# Run the task asynchronously
execution = client.task.run(
    task_id=task_id,
    version="1",
    async_=True,  # Enable async execution
    inputs={
        "ANCHOR_TARGET_URL": "https://example.com",
        "ANCHOR_MAX_PAGES": "10"
    }
)

print(f"Task execution started: {execution.data}")
# Response: {'async': True, 'success': True, 'message': 'Task execution started', 'taskId': '...'}
```
</CodeGroup>

### Checking Execution Results

After starting an async legacy task, you can check its execution status and results by querying the task's execution history endpoint:

<CodeGroup>
```typescript node.js
// Get execution results
const response = await fetch(`https://api.anchorbrowser.io/v1/task/${taskId}/executions?page=1&limit=1&version=1`, {
  headers: {
    'anchor-api-key': process.env.ANCHORBROWSER_API_KEY,
    'Content-Type': 'application/json'
  }
});

const results = await response.json();
console.log('Execution results:', results.data);
```

```python python
import os
import requests

# Get execution results
response = requests.get(
```

### Polling for Results

For async legacy tasks, you can implement polling to wait for completion:

<CodeGroup>
```typescript node.js
// Poll for task completion
async function waitForTaskCompletion(taskId: string, maxAttempts: number = 60) {
  for (let i = 0; i < maxAttempts; i++) {
    const response = await fetch(`https://api.anchorbrowser.io/v1/task/${taskId}/executions?page=1&limit=1`, {
      headers: {
        'anchor-api-key': process.env.ANCHORBROWSER_API_KEY,
        'Content-Type': 'application/json'
      }
    });
    
    const results = await response.json();
    const latestResult = results.data?.results?.[0];
    if (latestResult) {
      if (latestResult.status === 'success' || latestResult.status === 'failure') {
        return latestResult;
      }
    }
    
    // Wait 2 seconds before next poll
    await new Promise(resolve => setTimeout(resolve, 2000));
  }
  
  throw new Error('Task execution timeout');
}

// Usage
const execution = await client.task.run({
  taskId: taskId,
  version: '1',
  async: true,
  inputs: { ANCHOR_TARGET_URL: 'https://example.com' }
});

const result = await waitForTaskCompletion(taskId);
console.log('Task completed:', result);
```

```python python
import os
import time
import requests

# Poll for task completion
def wait_for_task_completion(task_id: str, max_attempts: int = 60):
    for _ in range(max_attempts):
        response = requests.get(
            f"https://api.anchorbrowser.io/v1/task/{task_id}/executions",
            params={"page": 1, "limit": 1},
            headers={
                "anchor-api-key": os.environ.get("ANCHORBROWSER_API_KEY"),
                "Content-Type": "application/json"
            }
        )
        
        results = response.json()
        latest_result = results['data']['results'][0] if results['data']['results'] else None
        if latest_result:
            if latest_result['status'] in ['success', 'failure']:
                return latest_result
        
        # Wait 2 seconds before next poll
        time.sleep(2)
    
    raise Exception('Task execution timeout')

# Usage
execution = client.task.run(
    task_id=task_id,
    version="1",
    async_=True,
    inputs={"ANCHOR_TARGET_URL": "https://example.com"}
)

result = wait_for_task_completion(task_id)
print(f"Task completed: {result}")
```
